# Конвенции проекта vkoxide

## 1. Концепция и Архитектура
- **Цель**: создать легковесную и удобную библиотеку (SDK) для работы с VK API на Rust.
- **Вдохновение**: Teloxide (Telegram) — удобный роутинг, типизированные обработчики, макросы, но в упрощенном виде.
- **Асинхронность**: Весь ввод/вывод выполняется асинхронно поверх `tokio`.

## 2. Стиль и структура кода
- **Обработка ошибок**: Внутри библиотеки (в `src/`) использовать `thiserror` для строгой типизации возвращаемых ошибок (`VkError`, `ApiError`). Использование `anyhow` допустимо только в `examples/` и конечных приложениях.
- **Типизация**: Ответы API типизируем с помощью `serde`. Неизвестные/новые структуры покрываем через fallback (например, `Unknown`), но стремимся к полному покрытию.
- **Язык**: Код, комментарии в `src/` и примеры остаются на английском для совместимости с экосистемой Rust, но документация и общение могут быть на русском. 
- **Модульность**:
  - `bot` — конфигурация и HTTP-клиент.
  - `dispatcher` — получение событий через LongPoll (или Webhooks).
  - `filters` — предикаты событий.
  - `types` — структуры данных VK API.
  - `prelude` — удобный импорт основных компонентов.

## 3. Обработка событий (Диспетчер)
- Разделение ответственности: LongPoll или Webhook отвечает только за получение `Update`.
- Простая система обработчиков (handlers), которая сопоставляет нужный тип события (`Message`, `Typing` и т.д.) с пользовательской функцией.

## 4. Развитие с AI
- Документ служит контрактом для генерации нового кода.
- Любое добавление новых методов API или типов событий сверяется с общим стилем (типизация -> интеграция -> проверка в `examples`).
